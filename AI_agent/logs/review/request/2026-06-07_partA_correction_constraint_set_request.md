## 2. 待审设计

### 2.1 切割轴 = 确定性 vs 判断（= 未来 codify 接缝）

不按"现象"（闭缝/粒度/常识/分级）切，改按**操作性质**切。这根轴 = 确定性 vs 判断 = 未来切配/idfpy 成熟时可整篇抬走变代码的接缝（确定性篇抬走，判断篇留 LLM）。

### 2.2 五篇分文档

| 篇 | 类型 | 管什么 | 对应 sm21 病灶 | 未来归宿 |
|---|---|---|---|---|
| **A0 容差与优先级契约** | 脊柱 | 容差常数(吸附栅格50mm/闭缝阈值/面积·WWR ±5%) + 数据优先级阶梯(一致测量>尺寸链>笔画估值>常识先验) + `corrections[]` 审计契约 + confidence 传递 | 全部引用它 | 常数表,不变 |
| **A1 坐标归一化** | 确定性 | 单一世界参照系 + 中线约定(化解内表面vs中线240mm偏移) + 立面 local→world | ③走廊x=0.24 | 可codify→几何内核前处理 |
| **A2 正则化/吸附** | 确定性(给定容差) | 规范轴集+跨层统一(段错杀手) + 栅格量化 + 子链=总长闭合 | ②跨层抖动碎片、④子链≠总长、⑥噪声 | 可codify→几何内核前处理 |
| **A3 冲突仲裁与补全** | 判断 | 超容差冲突的通道仲裁 + 缺失尺寸补全 + 不合理值兜底(幽灵房) | ①笔画vs链、⑤幽灵房 | 留LLM→zonification判断层 |

### 2.3 落地序（运行时序 ≠ 撰写优先级，分开）

- **运行时管线**：A1 → A2 → A3(+A4)（先归一参照系 → 再吸附正则 → 再仲裁补全）。
- **撰写/落地优先级**（ROI×安全×地基性）：
  1. **A0**（脊柱,没它没法写任何规则;逼出取舍决策;立起 `corrections[]` 字段供评测归因）
  2. **A2**（段错杀手:跨层轴统一直接干掉退化碎片;确定性、几乎无误修真实设计风险;ROI 最高)
  3. **A1**（A2 的确定性搭档;运行时在 A2 上游但崩溃优先级低,与 A2 同批)
  4. **A4(stub)→A3**（判断重、带"修掉真实异常设计"红线风险 → 压到 B2-B4 评测尺子立起来后再上)

## 3. 关注点（请重点挑战）

1. **切割轴对不对**：按"确定性 vs 判断"切，比按现象（闭缝/粒度/常识）或按数据通道（笔画/尺寸链/立面）或按几何维（x/y 平面 vs z 竖向）切，是否真更好？有无第三类操作被漏掉？
2. **五篇分解是否干净**：A0-A4 边界清不清？有无重叠/遗漏？A4(数据) 与 A3(规则) 分离、A0(常数脊柱) 单列——值不值？会不会太碎（5 篇对 skill 库是否过细）？
3. **"确定性篇=未来 codify 接缝"是否成立**：A1/A2 真能在切配/idfpy 成熟时整篇抬走变代码、A3/A4 真能原样留 LLM 吗？还是会纠缠？
4. **落地序合不合理**：A0→A2→A1→A4(stub)→A3。A2 先于 A1 写（崩溃优先）但运行时 A1 在前——这种"撰写序≠运行序"会不会埋坑？
5. **红线**：A4 常识先验严格作裁决/兜底、不覆盖一致测量数据（优先级 = 一致测量>尺寸链>先验）——这条够不够防"修掉真实 1.2m 设备间"？要不要更硬的机制？
6. **`corrections[]` 审计日志定为硬要求**对不对？（论点:没它=放弃可解释性与"看错 vs 改错"评测归因。）
7. **与 zonification 的精度耦合**：Codex zonification 调研指出 perimeter_core 模式**不需高内墙精度**（[../review/2026-06-07_zonification_approach_review.md](../review/2026-06-07_zonification_approach_review.md) 答6）。partA 把精度做满,是否对"主线=忠实建模(room_identity,需高精度)"成立、但相对"将来可能走 perimeter_core"过度投资？A1/A2 便宜确定性档安全、A3/A4 精度判断档是否该克制？
8. **phase1 配套小改**：phase1 别再吐"估算坐标冒充测量值",把笔画/尺寸链作两独立通道+置信度交出（recognition_modeling §4）。这条放 partA 请求里对不对,还是属 phase1 skill 另议？
